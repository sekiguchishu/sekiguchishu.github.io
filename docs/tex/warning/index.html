注意事項

　TeXを使ううえでの注意事項を，重要な順に列挙します。

ファイル名・フォルダ名は，半角英数字とアンダーバーだけにすべし

　ドットtexファイル本体の名前だけでなく，それに至るまでのフルパスにも，日本語は使ってはいけません。例えば，Windowsの場合，
“C:\Users\shu\Desktop\卒論\template\tex\sotsuron_main.tex”
のように，途中のフォルダ名に入っていてもだめです。また，図として挿入するPDFファイル名にも注意してください。日本語がだめなのはもちろん，「ドット」や「半角スペース」もだめです。ただし拡張子のドットは大丈夫です。

ベタ書き禁止，Figref，Tabref，Chaprefを使うべし

　TeXのソースコード内で，図~\ref{ラベル名}やTable.~\ref{ラベル名}のように直接書かずに，\Figref{ラベル名}や\Tabref{ラベル名}と書くようにしましょう。他にも，\Equref（式），\Chapref（章），\Secref（節），\Subsecref（項），\Subsubsecref（目）があります。

様式は自己責任で確認すべし

　論文のフォーマット，特に，表紙の様式などは，私が在籍していた当時と変わっているおそれもあります。ご自身でしっかり確認してください。

図のサイズは相対量で指定すべし

　図や表のサイズ，余白の大きさなどを調整する際，「mm」や「pt」などの絶対量で指定するのではなく，\textwidthや\columnwidthなどの相対量を使いましょう。

改行されないスペース「~」を使うべし

　数字と単位の間，もしくは変数と単位の間にはスペースが要るので，1.2 m，x [kg]のように書きます。ところがそうすると，そのスペースのところで改行されてしまうおそれがあり，1.2
mのように不格好になってしまいます。それを避けるために，改行されないスペースというものがあり，1.2~mのように書きます。

ベクターイメージとラスターイメージを理解すべし

　ベクターイメージとは，「この座標からこの座標に直線が通っています」のような，描画に関する情報のみを保存する画像データ形式です。それに対してラスターイメージとは，「このピクセルは赤，その隣のピクセルは青，…」というように，1点ごとに情報を保存する画像データ形式です。ベクターイメージとラスターイメージには，それぞれ得意不得意があります。たとえば実験結果のグラフのような画像は，基本的には線の通っている座標さえ分かれば再現できますから，ベクターイメージが適しています。対して，カメラで撮った写真のような画像はラスターイメージが適しています。またベクターイメージには，拡大しても画質が荒くならないという特長があります。以上をふまえて，基本的に，論文に載せるグラフなどは，ベクターイメージであるPDF形式で保存して，texに挿入しましょう。例外として，写真が含まれる場合は，PDF化しても意味が無いので，jpgやpngのまま挿入しましょう。

「./」と「../」の違いを理解すべし

　texファイルに画像を挿入する際，画像のパスを記述する必要があります。その際，そのtexファイルが保存されている位置から見た，相対パスを指定します。このとき「./」はカレントディレクトリ，つまりそのtexファイルと同じフォルダを指します。対して「../」はペアレントディレクトリ，つまりそのtexファイルが所属しているフォルダのひとつ上の階層のフォルダを指します。テンプレートでは，TeXソースファイルを格納する「tex」フォルダと，画像ファイルを格納する「fig」フォルダが同じ階層にあるので，texファイルから画像ファイルへの相対パスは，一度「../」によって階層をひとつ上がり，そこから「../fig/～」のように下がっていく，という書き方をしています。それと異なり，例えばtexファイルと同じ階層に「a.jpg」というファイルがあるなら「./a.jpg」のように指定します。

「h」と「H」の違いを理解すべし

　TeXに図を挿入するとき，その位置を制御するオプションはいくつかの種類があります。基本的には「h」を使っておけばよいのですが，どうしてもその位置に固定したいというときは「H」を使いましょう。

分割コンパイルすべし

　論文の執筆は，章ごと，あるいは節ごとにファイルを分けておこないましょう。テンプレートは章ごとになっています。各章のファイルも，単体でコンパイルできるようになっています。ただし注意点として，章番号は1になります。また，そのファイルの中に無い参照（citeや，別の章に対するSecrefなど）は，表示が??になります。1になったり??になったりせずに，本番通りの状況で確認したい場合は，mainをコンパイルしてください。ただし時間はかかります。

表ジェネレーターを使うべし

　TeXの欠点のひとつが，表を扱いづらいところです。特に，大規模な表をTeXのコマンドに沿って手入力するのは時間の無駄です。表ジェネレーターを使いましょう。Webで「tex 表ジェネレーター」で検索すればいくらでも出てきます。また，TeXの表は融通が利かないので，Excelで作った表をPDF化して挿入するのも手です。その場合，キャプションが「図 1.1」のようになってしまうので，\begin{table}の中に\includegraphicsを入れるという手法をとりましょう。

左側をパンチ穴で綴じるため，テンプレートでは，全体を少し右に寄せている

　電子データを提出する（修論のみ）際には，この右寄せは，無いほうがいいのかもしれません（私は電子データも右に寄せたまま提出しました）。

段落字下げのために全角スペースを入れるべからず

　TeXは自動で段落字下げをしてくれます。わざわざ全角スペースを入れないようにしましょう。段落を変えたければ，\parを使います。

「あるかもしれない」精神でググるべし

　TeXでやりたいと思ったことは，たいてい誰かがすでにやっています。TeXの各種コマンドは非常に豊富なので，やりたいことがあったらまずはググりましょう。また，TeXは，基本的な書き方さえ習得すれば，あとはその都度ググる，という方法でなんとかなります。

rmの使い方に注意すべし

　数式のドルマークの中で，単位を入れるときは，斜体ではなく立体にしたいです。そのときは，$1.0\times 10^3~\mathrm{[m]}$のように書きます。同様の用途で，\rmというコマンドがあるのですが，これは，以降をすべて立体にしてしまうので，\rm{[m]}ではなく，{\rm [m]}のように，中括弧の中で使います。
